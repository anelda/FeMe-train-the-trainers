Read `2-fact-checked.md` in the current directory and add a glossary of key terms. Write the result to `3-glossary.md`.

## Process

1. Read through the entire chapter
2. Identify all technical or domain-specific terms that may be new to the reader (e.g. "cognitive load", "formative assessment", "backward design", "scaffolding")
3. Include terms that are defined inline in the text — the glossary serves as a quick-reference collection
4. Add the glossary as a section at the end of the chapter, before any "Further Reading" section if one exists

## Glossary format

Use a definition-list style:

```
## Glossary

**Term** — A clear, plain-language definition in 1–2 sentences. Avoid circular definitions. If the term is used in a specific way in this workbook that differs from everyday usage, say so.

**Another term** — Definition here.
```

## Guidelines

- Use the **voice-and-tone** skill — definitions should be warm, direct, and jargon-free
- Aim for 8–20 terms depending on the chapter's density
- Sort terms alphabetically
- Do not include terms that are common knowledge for the target audience (educated professionals)
- Do not duplicate the full explanation from the chapter — keep definitions concise
- If a term appears in multiple chapters, still include it — each chapter's glossary should be self-contained
